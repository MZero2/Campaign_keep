# Security Policy

If you discover a security issue, please do not open a public issue with exploit details.

Instead, report it privately to the repository owner and include:

- a short description of the issue
- affected area
- reproduction steps
- potential impact

## Scope

This project is a portfolio application and is still evolving. Reasonable care is taken around:

- session handling
- secret management through environment variables
- role-aware access checks
- separation between public and private campaign content

That said, it should still be treated as a demo-stage codebase rather than a security-audited production platform.
